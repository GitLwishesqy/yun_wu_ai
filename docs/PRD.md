## 1. 产品概述

### 1.1 产品愿景

打造一款**真正具备教学能力的 AI 英语陪练智能体平台**，而非简单的 AI 聊天机器人。平台需要能够：

- **主动纠错**：识别语法、发音、用词错误并给出教学解释
- **自适应教学**：根据学习者水平动态调整难度和内容
- **长期陪伴**：记录学习轨迹，追踪薄弱点，规划学习路径
- **多模态交互**：覆盖听、说、读、写四项核心语言能力

### 1.2 目标用户

| 学段 | 年龄段 | 核心需求 | 对标体系 |
|------|--------|---------|---------|
| 小学 | 6-12岁 | 兴趣培养、基础发音、简单对话 | 新课标 1-3 级 / CEFR Pre-A1~A1 |
| 初中 | 12-15岁 | 中考口语、日常交流、基础写作 | 新课标 4-5 级 / CEFR A2~B1 |
| 高中 | 15-18岁 | 高考听说、学术表达、思辨写作 | 新课标 6-7 级 / CEFR B1~B2 |
| 成人 | 18岁以上 | 职场英语、雅思/托福、商务沟通 | CEFR B1~C2 / 雅思/托福对标 |

### 1.3 核心差异化

| 维度 | 普通 AI 聊天 | 本产品 (陪练智能体) |
|------|-------------|---------------------|
| 教学目标 | 无 | 明确的学习目标和能力评估 |
| 纠错 | 忽略或简单纠正 | 分级纠错 + 解释 + 练习 |
| 适应 | 无差别回复 | 根据水平动态调节 |
| 记忆 | 短期上下文 | 长期学习档案 + 薄弱点追踪 |
| 评测 | 无 | 多维度评分报告 |
| 场景 | 泛化 | 结构化场景库 (对标考纲) |

## 3. 功能需求详述

### 3.1 智能陪练核心引擎 (P0 — 核心)

#### 3.1.1 多场景口语陪练

**需求描述**：AI 驱动的角色扮演对话，支持预设场景和自由对话。

**功能要点**：

| ID | 功能 | 详细描述 | 验收标准 |
|----|------|---------|---------|
| F-001 | 场景选择 | 按学段和难度筛选场景模板 | 支持搜索、分类筛选、难度标签 |
| F-002 | 场景对话 | 在选定场景中与 AI 进行角色扮演对话 | 对话连贯性 > 20 轮；AI 角色扮演一致性不丢失 |
| F-003 | 自由对话 | 无预设场景的开放式英语对话 | 话题漂移后能自然引导回学习主线 |
| F-004 | 场景切换 | 对话中自然过渡到新场景 | 用户可手动切换，AI 也可建议切换 |
| F-005 | 对话摘要 | 每次陪练结束后生成对话摘要 | 包含：使用词汇、语法点、错误统计、改进建议 |

**场景库要求**：

- MVP 阶段：不少于 **15 个**结构化场景
- 正式版：不少于 **50 个**场景
- 每个场景须包含：场景名称、难度等级 (1-9)、学段标签、核心词汇列表 (10-20个)、目标句型 (3-5个)、角色设定描述、对话引导脚本

#### 3.1.2 智能纠错引擎

**需求描述**：实时或准实时地识别用户表达中的错误，分级纠正。

**功能要点**：

| ID | 功能 | 详细描述 | 验收标准 |
|----|------|---------|---------|
| F-006 | 语法纠错 | 识别语法错误并纠正 | 准确率 > 90%；给出中文解释 |
| F-007 | 发音评测 | 音素级发音评估 | 定位到具体音节/单词的发音问题 |
| F-008 | 用词建议 | 建议更地道/更高级的表达 | 分级建议：基础纠错/进阶提升 |
| F-009 | 纠错策略 | 智能选择纠错时机和方式 | 不打断流利度；关键错误必纠；轻微错误选择性纠 |
| F-010 | 错误学习卡 | 每种错误附带学习卡片 | 卡片内容：错误原文→正确版本→原因解释→同类练习 |

**纠错策略分级**：

```
┌─────────────────────────────────────────────┐
│              纠错决策树                       │
│                                             │
│  用户表达 ──→ 严重错误(影响理解)?             │
│                │                            │
│         ┌──────┴──────┐                     │
│        是              否                    │
│         │              │                    │
│    立即纠正         重复错误?                  │
│    + 解释              │                    │
│                   ┌────┴────┐               │
│                  是         否               │
│                   │          │              │
│              本轮结束时    记录但不打断        │
│              纠正+练习    下轮再观察          │
└─────────────────────────────────────────────┘
```

#### 3.1.3 自适应难度调节

**需求描述**：根据用户实时表现动态调整对话难度。

**功能要点**：

| ID | 功能 | 详细描述 | 验收标准 |
|----|------|---------|---------|
| F-011 | 初始定级 | 首次使用时通过测试对话定级 | 对标 CEFR (A1-C2) 和国内新课标 1-9 级 |
| F-012 | 动态调节 | 根据对话表现实时调级 | 连续3轮无错误→自动升0.5级；连续2轮同类错误→降0.5级 |
| F-013 | 语速控制 | TTS 语速自适应 | 初级 0.75x / 中级 1.0x / 高级 1.25x / 可手动调节 |
| F-014 | 词汇控制 | 自动控制生词比例 | 每轮对话生词比例控制在 5%-15% (可理解输入假说) |
| F-015 | 句长控制 | 根据水平控制AI输出句长 | 初级≤10词/句；中级≤20词/句；高级无限制 |

#### 3.1.4 长期学习记忆 (Learner Profile)

**需求描述**：构建用户学习档案，追踪学习历史和薄弱点。

**功能要点**：

| ID | 功能 | 详细描述 | 验收标准 |
|----|------|---------|---------|
| F-016 | 错误追踪 | 记录每次对话的错误类型和频率 | 可视化薄弱点雷达图 (语法/发音/词汇/流利度/逻辑) |
| F-017 | 词汇量估算 | 动态追踪已掌握/接触/未接触词汇 | 词汇量趋势图 |
| F-018 | 学习曲线 | 记录各维度能力变化趋势 | 支持日/周/月/学期粒度 |
| F-019 | 遗忘提醒 | 基于艾宾浩斯遗忘曲线推送复习 | 自动生成薄弱知识点的复习任务 |
| F-020 | 学习建议 | 基于数据分析生成个性化学习建议 | 每周更新一次学习建议 |

### 4.9 组件设计原则

1. **轻盈**：大面积留白，卡片使用浅绿底色 (primary-50/100)，避免沉重感
2. **柔和**：圆角 > 直角，阴影柔和，错误色使用柔红而非刺眼红色
3. **生机**：关键交互点使用嫩芽绿，搭配暖杏色点缀，传达成长感
4. **清晰**：信息层级分明，对话气泡左右分明，纠错卡片颜色区分
5. **包容**：支持深色模式 (深绿色基调)，符合无障碍 WCAG AA 标准
6. **品牌统一**：所有页面遵循同一色彩体系，避免割裂感

### 4.10 亲和力设计 (面向小学用户)

针对小学用户群体，在标准视觉基础上增加：

- **萌趣元素**：AI 角色拥有可爱的小动物/IP 形象 (如小猫头鹰、小鸟)
- **动画反馈**：答对时小动物跳跃、撒花；纠错时小动物温柔摇头
- **大字体**：默认字号比成人模式大 2px
- **简化布局**：减少信息密度，每屏只展示一个核心操作
- **语音优先**：低年级用户默认语音输入，减少打字负担
- **色彩更活泼**：在淡绿基础上增加更多暖杏色和樱花粉的点缀

## 7. 安全合规规范

### 7.1 内容安全

| 层级 | 措施 | 说明 |
|------|------|------|
| **输入层** | 敏感词过滤 + 意图检测 | 用户输入先过敏感词库，检测到违规内容→拒绝处理 |
| **模型层** | System Prompt 约束 + 知识边界限定 | 每套 Prompt 明确禁止生成的内容类型；限定知识范围 |
| **输出层** | 内容审核 API + 规则引擎 | LLM 输出先过审核 API(阿里云/易盾)，不合规内容替换或拦截 |
| **兜底层** | 人工复审 + 快速封禁 | 高风险内容进入人工复审队列；确认违规→封禁用户 |

**审核流水线**：

```
用户输入 → [敏感词检测] → [意图识别] → LLM处理 → [输出审核] → [兜底替换] → 返回用户
                │              │                        │
                ▼              ▼                        ▼
           拒绝+警告      标记风险             人工复审队列
```

### 7.2 数据安全

| 领域 | 措施 |
|------|------|
| 传输安全 | 全站 HTTPS (TLS 1.3)；API 签名验证 |
| 存储安全 | 密码 bcrypt 加盐哈希；语音文件 AES-256 加密；数据库字段级加密 (手机号等) |
| 访问控制 | RBAC 权限模型；API 级权限校验；数据行级隔离 (家长只能看自己孩子) |
| 数据最小化 | 仅收集必需信息；语音数据定期清理；用户可导出/删除数据 |
| 备份恢复 | 数据库每日自动备份；异地容灾；RTO < 4小时 |

### 7.3 合规要求清单

| 法规/标准 | 关键要求 | 状态 |
|-----------|---------|:----:|
| 《个人信息保护法》 | 隐私政策、知情同意、数据权利 (查阅/更正/删除/携带) | 待实现 |
| 《未成年人保护法》 | 未成年人模式、时长限制、消费限制 | 待实现 |
| 《儿童个人信息网络保护规定》 | <14岁儿童信息需监护人同意、最小收集、专门保护 | 待实现 |
| 《生成式人工智能服务管理暂行办法》 | AI生成内容标识、内容审核、投诉举报机制、算法备案 | 待实现 |
| 教育 APP 备案 | 教育部教育移动互联网应用程序备案 | 待实现 |
| 等保二级/三级 | 网络安全等级保护测评 | 待实现 |
| ICP 备案 | 网站/APP 上线必备 | 待实现 |
| 《数据安全法》 | 数据分类分级、安全保护义务 | 待实现 |

### 7.4 AI 模型安全

| 攻击类型 | 防护措施 |
|---------|---------|
| **Prompt Injection** | 用户输入与系统指令分层隔离；输入清洗去特殊标记；System Prompt 注入防御指令 |
| **Jailbreak** | 定期红队测试；异常行为检测 (用户反复尝试绕过)；快速响应封禁 |
| **数据投毒** | 训练/微调数据审核；RAG 知识库内容白名单审核 |
| **Token 消耗攻击** | 单用户输入长度限制 (2000字符)；单次对话轮次上限 (50轮)；单用户日 Token 消耗上限 |
| **模型幻觉** | RAG 事实核查；知识边界声明；不确定性标记 |
| **隐私泄露** | 用户数据不用于模型训练 (默认)；API 调用签署 DPA；本地化部署方案备选 |
